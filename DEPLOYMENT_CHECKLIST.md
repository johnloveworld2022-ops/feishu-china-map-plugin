## 🚀 部署步骤

### 1. 最终构建


### 2. 验证构建产物
- 检查 `dist/` 目录内容完整
- 确认资源路径为相对路径
- 验证地图数据文件存在

### 3. 提交到GitHub
```bash
git add .
git commit -m "feat: 准备飞书插件部署版本"

```

- 在飞书开发者后台提交GitHub仓库链接
- 指定部署目录为 `dist`
- 填写插件信息和权限申请

## 📝 注意事项

### 环境差异
- **开发环境**: 使用代理服务器和模拟数据
- **生产环境**: 直接使用飞书SDK和API
- **自动检测**: 代码会自动检测环境并选择合适的调用方式

### 依赖管理
- **ECharts**: 通过CDN加载，有多个备选源
- **地图数据**: 本地文件，包含在构建产物中
- **飞书SDK**: 由飞书环境提供

### 功能限制
- 当前版本专注于数据可视化
- 仅读取数据，不修改多维表格内容
- 符合最小权限原则
